# Portfolio Website — Project Instructions

## Owner
Chris-Esezobor Ejodamen

## Stack
- React 18 + Vite
- Plain CSS (no CSS-in-JS, no Tailwind)
- No external UI libraries

## Design Rules
- Colour palette: deep warm black (`#0d0906`), burnt orange (`#c4581c`), blush pink (`#c9837a`)
- Typography: Space Grotesk (headings), Inter (body)
- No emojis anywhere in the UI
- Minimal animations only — no particles, floating elements, or looping background effects
- Buttons in hero section must remain pill-shaped (`border-radius: 50px`), black fill, white on hover

## Code Rules
- Treat this project as a practice project — the user wants to type out most logic manually and receive detailed explanations of what each line or block does under the hood
- No comments unless the logic is genuinely non-obvious
- No new dependencies without explicit approval
- Keep components in `src/components/`, one `.jsx` + one `.css` per component
- Global tokens (colours, spacing, typography, keyframes) live in `src/index.css` only
- Before every task: suggest whether the user should type it manually (yes for new concepts worth learning, no for repetitive boilerplate)

## Git Rules
- Never include Co-Authored-By Claude unless explicitly instructed
- Never run git commands on behalf of the user — always provide the command and let them type it themselves so they learn it
- Only explain what a git command does when asked
